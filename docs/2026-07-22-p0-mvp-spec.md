# Lumo P0 MVP Spec

> 日期：2026-07-22
> 状态：已确认，待写实现计划

## 概述

P0 MVP 包含 7 个模块，目标是交付可内测的 Android 学习教练应用。核心逻辑用 Python（Senza），UI 用 Kotlin + Jetpack Compose，通过 Chaquopy 桥接。

---

## 模块 1：AI 导师对话

### 功能

- 流式对话输出，Kotlin Compose 渲染 Markdown
- 多轮对话历史，持久化到 SQLite
- 对话历史可搜索（FTS5）、可删除
- 跨会话上下文记忆：通过 `read_memory` / `write_memory` 工具自动读写学习者画像，新会话能记得之前学过什么、哪里薄弱。不引入独立 memory workflow
- 对话中 AI 可搜索笔记（`search_notes`）和错题本（`search_quiz_errors`）作为上下文
- 快捷提问：在学习任务详情页和对话页面内都有快捷按钮（讲解一下、举个例子、出几道题、我不理解），知识点来源包括当前任务关联的知识点和对话上下文

### Markdown 渲染要求（全部）

- 基础 Markdown：标题、列表、粗体/斜体、引用、分隔线
- 代码块：语法高亮 + 一键复制 + 横向滚动 + 可折叠
- 表格
- 数学公式：LaTeX `$...$` 行内 / `$$...$$` 块级，KaTeX 渲染
- Mermaid 图表
- 图片：URL + 本地
- 链接：可点击跳转
- 内联代码

### Python 侧

- `lumo.agent`：Senza `AgentHarness` 实现流式对话，挂载 4 个工具
- `lumo.tools`：`read_memory` / `write_memory` / `search_notes` / `search_quiz_errors`
- `lumo.store`：对话写入 `sessions` / `messages` 表
- 流式：Senza `stream_prompt` async generator → Chaquopy callback → Kotlin Flow

### Kotlin 侧

- 对话列表页（会话搜索、删除）
- 对话详情页（流式渲染、快捷提问按钮）
- Markdown 渲染组件（复用于笔记模块）

## 模块 2：学习目标 & 计划生成

### 功能

- 自然语言输入目标（如"2 个月学会前端基础，每天 1 小时"）
- 用户可同时有多个学习计划（如"前端基础"和"英语口语"并行）
- AI 四步 workflow：`decompose`（生成全部周大纲）→ `detail`（细化当前周日任务）→ `verify`（检查任务合理性）→ `publish`（写入 SQLite）
- 时间跨度策略：周大纲 + 逐周细化。decompose 生成完整周期大纲，detail 只细化当前周
- 计划可视化：时间轴展示整体进度
- 手动调整：排序、跳过、延后、编辑、新增、删除任务
- 计划可中断恢复：Foreground Service + Senza `WorkflowEngine.with_task_store`

### 记忆策略

- 混合：学习风格全局共享（跨所有计划），知识点掌握度按计划隔离

### Python 侧

- `lumo.workflows`：计划生成 workflow 定义（四步），Senza `WorkflowEngine` 执行
- `lumo.store`：`plans` / `tasks` 表 CRUD
- 流式进度：workflow 事件通过 callback 推送

### Kotlin 侧

- 目标输入页
- 计划列表页（多计划切换）
- 时间轴计划展示页
- 任务编辑（拖拽排序、跳过、延后、编辑、新增、删除）

## 模块 3：每日学习任务流

### 功能

- 首页今日任务看板，聚合所有 active plan 的当日任务
- 按 plan 分组展示，section 内按任务顺序排列
- 任务状态：待学 / 进行中 / 已完成 / 待复习
- "待复习"由错题触发：答错的知识点关联任务自动标记为"待复习"
- 内置番茄钟：可配置时长，预设档位（25+5、50+10、15+5）
- 打卡机制：完成当日任务自动打卡
- 学习时长自动统计

### Python 侧

- `lumo.store`：任务 CRUD、状态流转、番茄钟计时逻辑、时长记录、打卡记录、Streak 计算

### Kotlin 侧

- 今日任务看板（按 plan 分组）
- 番茄钟 UI（计时显示、开始/暂停/停止）
- 打卡展示

## 模块 4：智能测验 & 错题本

### 功能

- 触发方式：任务完成后提示"是否做几道练习题？" + 用户手动在题库页面触发
- AI 生成 3-5 道题，题型：单选、多选、判断、简答
- 测验 workflow：`generate` → `verify` → `publish`
- 即时判分：客观题（单选/多选/判断）直接判，简答题 AI 判分
- AI 详细解析
- 错题自动收录，标注错误原因
- 答题可中断：逐题记录，退出后下次继续

### Python 侧

- `lumo.workflows`：测验生成 workflow，Senza `WorkflowEngine`
- `lumo.agent`：简答题 AI 判分
- `lumo.store`：`quiz_questions` / `quiz_answers` 表 CRUD

### Kotlin 侧

- 答题界面（各题型渲染、进度显示）
- 错题本（按知识点分类、错误原因展示）
- 答题中断恢复

## 模块 5：学习笔记

### 功能

- 笔记来源：
  1. 计划生成时自动创建空白笔记（关联知识点）
  2. 对话内容 AI 总结为笔记（对话页面"保存为笔记"按钮）
  3. 用户粘贴文本导入
- AI 一键生成知识点摘要
- Markdown 编辑（标题、列表、代码块）
- 文件夹分类管理
- 笔记渲染复用模块 1 的完整 Markdown 渲染组件（代码高亮、LaTeX、Mermaid 等全部能力）

### Python 侧

- `lumo.agent`：对话总结为笔记、AI 知识点摘要生成
- `lumo.store`：`notes` / `folders` 表 CRUD + FTS5 搜索

### Kotlin 侧

- 笔记列表（文件夹树 + 笔记列表）
- Markdown 编辑器
- "保存为笔记"按钮（对话页面内）

## 模块 6：学习数据统计

### 功能

- 打卡热力图（日历视图，按月展示）
- 连续学习天数 Streak
- 日 / 周 / 月 / 全部学习时长趋势
- 知识点掌握度进度条
- 全部历史数据，可切换视图

### Python 侧

- `lumo.store`：数据聚合查询（打卡记录、学习时长、知识点掌握度）

### Kotlin 侧

- 热力图日历组件
- Streak 展示
- 趋势图表（日/周/月/全部切换）
- 知识点掌握度进度条

## 模块 7：模型配置

### 功能

- 用户配置单个 LLM provider + API Key，全局使用
- 支持多家：OpenAI、Anthropic、DeepSeek、通义、豆包
- 可自定义 base_url（兼容开源模型 API）
- 连接测试 + 用量估算

### Python 侧

- `lumo.agent`：Senza provider 创建（`create_openai_provider` / `create_anthropic_provider`），连通性验证
- `lumo.store`：`settings` 表存储配置

### Kotlin 侧

- 配置表单（provider 选择、API Key 输入、base_url 输入）
- 连接测试按钮 + 结果反馈
- 用量估算展示

## 技术风险

1. **Senza 在 Android 上通过 Chaquopy 运行** — Senza 是 PyO3 扩展模块，需验证 Chaquopy 能否加载含原生 Rust 扩展的 wheel。这是最高优先级的技术验证
2. **Senza `stream_prompt` async generator 通过 Chaquopy 桥接** — 需验证 Python async generator 能否通过 Chaquopy callback 推送到 Kotlin Flow
3. **iOS Python 嵌入** — 后续阶段，通过 PyOxidizer 静态编译或参考 Pyto 方案
